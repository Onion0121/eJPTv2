# Introduction

Metasploit Routing is a pivoting technique that allows penetration testers to use a compromised machine as a gateway to access internal networks through Meterpreter sessions.

When an attacker compromises a host, that machine may have access to networks that are not reachable directly.

Metasploit can create routes through the compromised system, allowing other modules to communicate with internal targets.

Example:

```
Attacker Machine

↓

Compromised Host

↓

Internal Network

↓

Hidden Targets
```

---

# What is Metasploit Routing?

Metasploit Routing allows the framework to send traffic through an active Meterpreter session.

Example environment:

```
Kali Linux

192.168.1.50


↓

Windows Server

192.168.1.100
10.10.10.5


↓

Internal Server

10.10.10.20
```

The attacker cannot directly reach:

```
10.10.10.20
```

Metasploit routing allows access through:

```
192.168.1.100
```

---

# Why Metasploit Routing Matters

Enterprise networks are usually segmented.

Example:

```
Internet

↓

Public Web Server

↓

Internal Network

↓

Database Servers
```

A compromised public server can become a bridge into the internal environment.

Metasploit routing allows testers to:

- Scan internal networks
- Run exploits against hidden hosts
- Enumerate services
- Perform lateral movement

# Adding Routes With Autoroute

Metasploit includes an autoroute script.

Background the session:

```text
background
```

Run:

```text
run autoroute -s 10.10.10.0/24
```

Explanation:

```
-s

Add subnet
```

Example:

```text
run autoroute -s 10.10.10.0/24
```

Creates a route:

```
Kali

↓

Meterpreter Session

↓

10.10.10.0/24
```

# Using Modules Through Routes

Once a route exists, Metasploit modules can target internal machines.

Example:

Search scanner:

```text
search portscan
```

Use:

```text
use auxiliary/scanner/portscan/tcp
```

Configure:

```text
set RHOSTS 10.10.10.20

set PORTS 22,80,445
```

Run:

```text
run
```

Traffic:

```
Scanner

↓

Meterpreter Pivot

↓

Internal Target
```

# Port Forwarding With Meterpreter

Meterpreter can forward ports.

Example:

Internal service:

```
10.10.10.20:80
```

Create local tunnel:

```text
portfwd add \
-l 8080 \
-p 80 \
-r 10.10.10.20
```

Now:

```
localhost:8080
```

connects to:

```
10.10.10.20:80
```

---

# Viewing Port Forwards

Command:

```text
portfwd list
```

Example:

```
Local Port

8080


Remote Host

10.10.10.20

Remote Port

80
```

---

# Removing Port Forwards

Remove:

```text
portfwd delete -l 8080
```

# SOCKS Proxy With Metasploit

Metasploit can create a SOCKS proxy.

Start:

```text
use auxiliary/server/socks_proxy
```

Configure:

```text
set SRVPORT 9050
```

Run:

```text
run
```

Creates:

```
SOCKS5

127.0.0.1:9050
```

# Troubleshooting

## Route Added But Scan Fails

Check:

```text
route print
```

Verify:

- Correct subnet
- Active session
- Target reachable

---

## Meterpreter Session Closed

Routes depend on sessions.

If the session dies:

```
Route disappears
```

Create a new session.

---

## Proxychains Problems

Check:

```bash
cat /etc/proxychains.conf
```

Verify:

```
socks5 127.0.0.1 9050
```

# Defense & Mitigation

Prevent Metasploit pivoting by:

- Segmenting networks
- Monitoring unusual connections
- Limiting host permissions
- Detecting Meterpreter behavior
- Restricting internal communication
- Applying endpoint protection
