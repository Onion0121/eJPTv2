# Introduction

This practical lab focuses on identifying and exploiting insecure FTP configurations, specifically anonymous FTP access.

FTP (File Transfer Protocol) is a network service used to transfer files between systems. When incorrectly configured, FTP servers may expose sensitive information or allow unauthorized users to access files without authentication.

Anonymous FTP allows users to connect without having a valid account on the system.

During this lab, you will practice:

- Discovering FTP services
- Identifying anonymous access
- Enumerating directories
- Downloading files
- Searching for sensitive information
- Understanding insecure FTP configurations

# FTP Fundamentals

FTP is a protocol designed for transferring files between computers.

FTP commonly uses:

Control channel:

```
Port 21/TCP
```

The control channel handles:

- Authentication
- Commands
- Session management

Data channel:

```
Dynamic ports
```

The data channel is used for:

- Directory listings
- File downloads
- File uploads

---

# Reconnaissance

Begin with a service scan:

```bash
nmap -sV -sC TARGET
```

Example:

```
21/tcp open ftp
```

The result indicates that an FTP service is running.

---

# Service Enumeration

Identify the FTP version:

```bash
nmap -sV -p21 TARGET
```

Example:

```
21/tcp open ftp vsftpd 3.0.3
```

Version detection can reveal:

- Software information
- Possible vulnerabilities
- Default configurations

# Identifying FTP

Connect manually:

```bash
ftp TARGET
```

Example:

```
Connected to TARGET

Name:
```

Try anonymous authentication:

Username:

```
anonymous
```

Password:

```
anonymous
```

or:

```
ftp
```

Successful login:

```
230 Login successful.
```

This confirms anonymous FTP access.

---

# Anonymous FTP Authentication

Anonymous FTP normally uses:

Username:

```
anonymous
```

Password:

```
anonymous
```

or an email address:

```
test@test.com
```

If accepted, the server allows access without a real account.

This is considered a security risk if sensitive files are exposed.

# Downloading Files

Download a file:

```ftp
get filename
```

```ftp

```

```ftp

```

After downloading:

```bash
ls
```

```bash
cat filename
```


# Searching For Sensitive Information

Search downloaded files:

```bash

```

Search for:

```
username

password

database

secret

key

token
```

Analyze binary or unknown files:

```bash
strings filename
```

Sensitive information may include:

- User credentials
- Database passwords
- Configuration files
- SSH keys

# Exploitation Opportunities

Anonymous FTP access may lead to several attack paths.

## Information Disclosure

Example:

```
backup.sql
```

May contain:

```
Database credentials
```

---

## Credential Discovery

Example:

```
users.txt
```

Contains:

```
admin:password123
```

Credentials can be tested against:

- SSH
- FTP
- Web applications
- SMB services

---

## File Upload Abuse

Some FTP servers allow anonymous uploads.

Test:

```ftp
put test.txt
```

If successful, the server may have:

- Weak permissions
- Dangerous upload configuration
- Potential interaction with a web server

# Post Exploitation

If credentials are discovered, test them on other services.

SSH:

```bash
ssh user@TARGET
```

SMB:

```bash
smbclient -U user //TARGET/share
```

Web applications:

```
http://TARGET/login
```

After gaining access:

Check identity:

```bash
whoami
```

Check permissions:

```bash
id
```

Enumerate system:

```bash
uname -a
```
