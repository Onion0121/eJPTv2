# Introduction

Before learning penetration testing, you need a safe environment where you can attack vulnerable systems without affecting production networks.

A virtual lab allows you to:

- Practice whenever you want
- Break systems without consequences
- Reset machines instantly
- Learn networking
- Simulate real-world penetration tests

Building a proper lab is one of the best investments you can make during your cybersecurity journey.

---

# Why Build a Home Lab?

A penetration tester learns by doing.

Reading documentation alone is not enough.

A home lab allows you to practice:

- Network discovery
- Enumeration
- Exploitation
- Privilege escalation
- Pivoting
- File transfers
- Reverse shells
- Active Directory (later certifications)

The more time you spend inside your lab, the more comfortable you'll become during the eJPT exam.

# VirtualBox vs VMware

Both platforms are excellent choices.

## VirtualBox

### Advantages

- Free
- Open Source
- Cross-platform
- Easy to configure

### Disadvantages

- Slightly slower
- Lower graphics performance

---

## VMware Workstation

### Advantages

- Better performance
- Better networking
- Stable snapshots
- Excellent hardware support

### Disadvantages

- Commercial license (Workstation Pro is free for personal use at the time of writing, but licensing may change.)

---

For this guide, either virtualization platform works perfectly.

# Creating Your Lab

Example topology:

```text
                     Internet
                         │
                Windows Host
                         │
        ┌────────────────┴────────────────┐
        │                                 │
    VirtualBox / VMware Hypervisor
        │
 ┌──────┴───────────────┬──────────────┐
 │                      │              │
 │                      │              │
 ▼                      ▼              ▼

 Kali Linux        Windows 10     Metasploitable 2

192.168.56.10    192.168.56.20     192.168.56.30
```

All virtual machines should be connected to the same isolated virtual network.

This allows the attacker to reach every target while keeping the lab separate from your home network.

# Installing VirtualBox Guest Additions

Guest Additions improve the usability of your virtual machines.

Benefits include:

- Better screen resolution
- Shared clipboard
- Mouse integration
- Shared folders
- Drag & Drop support

Install Guest Additions after the operating system installation is complete.

---

# Shared Folders

Shared folders make file transfers easier.

Examples:

```
Host

C:\Labs

↓

Virtual Machine

/mnt/shared
```

Useful for:

- Payloads
- Wordlists
- Reverse shells
- Scripts
- Notes

Avoid relying on shared folders during realistic attack simulations.

# Snapshots

Snapshots are one of the most valuable virtualization features.

Take snapshots:

- Before installing software
- Before exploitation
- Before privilege escalation
- Before malware testing

Example:

```
Fresh Install

↓

Snapshot

↓

Practice

↓

Break Everything

↓

Restore Snapshot

↓

Repeat
```

This allows unlimited practice without reinstalling operating systems.

# Testing Network Connectivity

Before starting any lab:

Verify the machines can communicate.

On Windows:

```powershell
ping 192.168.56.10
```

On Kali:

```bash
ping 192.168.56.20
```

Check routing:

```bash
ip route
```

Check IP address:

```bash
ip addr
```

Windows:

```powershell
ipconfig
```

Confirm that every machine belongs to the same virtual network.

# Common Problems

## No Internet

Verify:

- NAT Adapter
- DHCP
- Default Gateway

---

## Machines Cannot Ping Each Other

Check:

- Network Adapter
- Firewall
- Host-Only configuration
- IP addresses

---

## Kali is Slow

Increase:

- RAM
- CPU cores
- Video Memory

Enable:

- 3D Acceleration

---

## Clipboard Doesn't Work

Install:

Guest Additions (VirtualBox)

or

VMware Tools.

---

## Mouse Gets Captured

Install Guest Additions or VMware Tools to improve pointer integration.

# Best Practices

- Keep your host operating system updated.
- Take snapshots regularly.
- Use strong passwords inside your lab.
- Store your notes in a dedicated folder.
- Learn to troubleshoot networking issues.
- Avoid exposing vulnerable virtual machines directly to the Internet.
- Back up important virtual machines.
- Label snapshots clearly.
- Keep enough free disk space for snapshots and additional labs.
