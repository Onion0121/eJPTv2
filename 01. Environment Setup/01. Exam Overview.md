# Introduction

The **eLearnSecurity Junior Penetration Tester (eJPT)** is an entry-level practical penetration testing certification designed to validate the skills required to perform a real-world security assessment.

Unlike multiple-choice certifications, the eJPT evaluates your ability to identify, enumerate, exploit, and document vulnerabilities in a controlled lab environment.

The exam focuses on methodology rather than memorization. Success depends on understanding how to approach a penetration test from start to finish.

---

# What is the eJPT?

The eJPT is a hands-on penetration testing certification that measures your ability to:

- Perform reconnaissance
- Enumerate hosts and services
- Identify vulnerabilities
- Exploit Windows and Linux systems
- Perform web application testing
- Escalate privileges
- Pivot through internal networks
- Gather evidence from compromised hosts

Rather than asking theoretical questions, the exam requires you to interact with live systems and solve realistic scenarios.

---

# Who is this Certification for?

The eJPT is ideal for:

- Beginners in Cybersecurity
- Students learning Ethical Hacking
- Help Desk or System Administrators moving into Security
- SOC Analysts transitioning to Red Team
- CTF players wanting structured penetration testing experience

Previous penetration testing experience is helpful but not required.

# Exam Format

The eJPT is entirely practical.

You will receive access to a lab environment containing multiple systems.

Your objective is to perform a complete penetration test by:

1. Discovering hosts
2. Enumerating services
3. Identifying vulnerabilities
4. Exploiting systems
5. Escalating privileges
6. Pivoting where necessary
7. Collecting evidence
8. Answering exam questions

Unlike a Capture The Flag competition, the goal is to demonstrate proper penetration testing methodology.

---

Service Enumeration

      │
      ▼
Post Exploitation
      │
      ▼
Pivoting

      ▼

Avoid skipping steps.

Most failures happen because students rush directly into exploitation without proper enumeration.

# Mindset During the Exam

The biggest mistake beginners make is trying random exploits.

Instead:

- Enumerate everything.
- Take notes.
- Read tool output carefully.
- Validate every assumption.
- Think methodically.

Remember:

> Enumeration wins exams.

---

# Time Management

Avoid spending hours attacking a single service.

A good strategy is:

- Initial Recon
- Full Nmap Scan
- Enumerate every service
- Identify low-hanging fruit
- Exploit easiest targets first
- Reuse credentials whenever possible
- Revisit difficult machines later

Working systematically is more important than working quickly.

---

# Common Mistakes

Avoid these mistakes:

- Running exploit after exploit without enumeration
- Ignoring UDP services
- Forgetting anonymous FTP access
- Missing SMB shares
- Not checking default credentials
- Forgetting to search for hidden directories
- Ignoring HTTP headers
- Forgetting privilege escalation
- Forgetting to pivot into internal networks
- Poor note-taking

---

# Preparation Tips

Before scheduling the exam, you should be able to:

- Enumerate common services confidently
- Perform Nmap scans without looking up commands
- Use Metasploit efficiently
- Generate payloads with Msfvenom
- Upload and stabilize reverse shells
- Perform Linux privilege escalation
- Perform Windows privilege escalation
- Enumerate web applications using Burp Suite
- Attack WordPress installations
- Crack password hashes
- Pivot through internal networks

If you can perform these tasks comfortably, you are likely ready for the eJPT.
