# Introduction

Server Message Block (SMB) is one of the most important services encountered during penetration tests and appears frequently in the **eJPT** exam.

SMB is used for:

- File sharing
- Printer sharing
- Remote administration
- Authentication
- Active Directory communication

Poorly configured SMB services often expose:

- Anonymous shares
- Sensitive documents
- Credentials
- Backup files
- User information

Enumeration is usually enough to obtain the initial foothold.

---

# What is SMB?

SMB is Microsoft's file sharing protocol.

Typical resources include:

```
Shared folders

Printers

Administrative shares

Domain resources
```

Common shares:

```
ADMIN$

C$

IPC$

NETLOGON

SYSVOL

Users

Backups
```

# Common SMB Ports

| Port | Protocol | Description |
|------|----------|-------------|
| 139 | NetBIOS | Legacy SMB |
| 445 | TCP | Modern SMB |

Verify with Nmap:

```bash
nmap -p139,445 TARGET
```

# Nmap Enumeration

Basic scan:

```bash
nmap -sV -p445 TARGET
```

Run SMB scripts:

```bash
nmap --script smb-enum-shares TARGET
```

Users:

```bash
nmap --script smb-enum-users TARGET
```

OS discovery:

```bash
nmap --script smb-os-discovery TARGET
```

Run all common SMB scripts:

```bash
nmap --script smb* TARGET
```

Example:

```
445/tcp open microsoft-ds

Windows Server 2019

Anonymous login allowed

Shares discovered
```

# Anonymous Access

Always test anonymous authentication.

List shares:

```bash
smbclient -L //TARGET/ -N
```

If successful:

```
Sharename

IPC$

Public

Backups
```

Anonymous access is a very common eJPT finding.

---

# Using smbclient

Connect anonymously:

```bash
smbclient //TARGET/Public -N
```

Authenticate:

```bash
smbclient //TARGET/Users -U administrator
```

Useful commands:

List files:

```
ls
```

Download:

```
get filename.txt
```

Upload:

```
put shell.exe
```

Current directory:

```
pwd
```

Change directory:

```
cd Documents
```

Recursive download:

```
recurse ON

prompt OFF

mget *
```

# Using enum4linux

One of the best SMB enumeration tools.

Run:

```bash
enum4linux TARGET
```

Common options:

Users:

```bash
enum4linux -U TARGET
```

Shares:

```bash
enum4linux -S TARGET
```

Passwords policy:

```bash
enum4linux -P TARGET
```

OS information:

```bash
enum4linux -o TARGET
```

Everything:

```bash
enum4linux -a TARGET
```

Typical output:

```
Users

Administrator

Guest

Backup

John
```

# Using enum4linux-ng

A newer alternative.

Run:

```bash
enum4linux-ng TARGET
```

- Users
- Groups

# Using CrackMapExec

Quick SMB assessment.

Anonymous:

```bash
crackmapexec smb TARGET
```

Authenticate:

```bash
crackmapexec smb TARGET \
-u administrator \
-p Password123
```

Check multiple hosts:

```bash
crackmapexec smb targets.txt
```

Enumerate shares:

```bash
crackmapexec smb TARGET --shares
```

Enumerate users:

```bash
crackmapexec smb TARGET --users
```

Enumerate sessions:

```bash
crackmapexec smb TARGET --sessions
```

Recovered credentials frequently work on:

# Common Misconfigurations

Common findings include:

- Anonymous shares
- Weak passwords
- World-readable backups
- Passwords stored in text files
- Old configuration backups
- Excessive permissions
- Guest access enabled
- Shared administrator documents

These issues are common in beginner labs and the eJPT exam.

---

# Defense & Mitigation

To secure SMB:

- Disable SMBv1
- Restrict anonymous access
- Use strong passwords
- Apply least privilege
- Remove unnecessary shares
- Monitor SMB activity
- Restrict firewall access
- Keep systems updated
- Audit permissions regularly

# Summary

SMB is one of the richest sources of information during a penetration test. Proper enumeration can reveal users, shares, credentials, and sensitive files without exploiting a single vulnerability.

In this chapter, you learned how to:

- Identify SMB services
- Enumerate shares
- Test anonymous access
- Download files
- Enumerate users and groups
- Use enum4linux and enum4linux-ng
- Use CrackMapExec
- Enumerate through RPC
- Search for credentials
- Reuse discovered credentials

Mastering SMB enumeration is essential for the **eJPT** exam, as SMB is one of the most frequently encountered services in both standalone Windows systems and Active Directory environments.
