# Introduction

Windows privilege escalation is the process of gaining higher privileges after obtaining access to a Windows system.

During a penetration test, attackers commonly obtain initial access through:

- Vulnerable services
- Weak credentials
- Web applications
- Phishing simulations
- Exploited software

The initial account is usually a low-privileged user.

The objective:

```
Low Privileged User

↓

Administrator / SYSTEM
```

Obtaining SYSTEM privileges provides complete control over the Windows machine.

---

# What is Windows Privilege Escalation?

Privilege escalation occurs when a user abuses:

- Misconfigured services
- Weak file permissions
- Vulnerable applications
- Excessive privileges
- Stored credentials
- Registry weaknesses
- Windows vulnerabilities

to obtain higher permissions.

Common privilege levels:

```
Guest

↓

User

↓

Administrator

↓

SYSTEM
```

---

# Windows User Privileges

Windows uses accounts and groups to control access.

Important accounts:

## Administrator

The local administrator account has extensive permissions.

Capabilities:

- Install software
- Modify system settings
- Manage users
- Access protected files

---

## SYSTEM

The highest local privilege.

SYSTEM can:

- Access almost all files
- Control services
- Modify security settings
- Manage processes

Many privilege escalation attacks aim for:

```
NT AUTHORITY\SYSTEM
```

---

## Normal Users

Standard accounts have limited permissions.

Example:

```
User
```

They cannot normally:

- Install drivers
- Modify protected directories
- Access administrator data

---

# Windows Security Model

Important components:

## Access Control Lists (ACL)

ACLs define who can access resources.

Example:

```
File

↓

Owner

↓

Permissions
```

Permissions include:

- Read
- Write
- Execute
- Modify
- Full Control

---

## Security Identifiers (SID)

Windows identifies accounts using SIDs.

Example:

```
S-1-5-21-...
```

Important:

Administrator:

```
RID 500
```

Guest:

```
RID 501
```

# File and Permission Enumeration

Windows permissions are controlled through ACLs.

Check permissions:

```cmd
icacls C:\path\file
```

Example:

```
User:(F)
```

Meaning:

```
Full Control
```

---

Find writable directories:

```cmd
icacls C:\ /grant
```

Look for:

- Writable service files
- Writable program folders
- Sensitive files


# Common Windows PrivEsc Paths

## Weak Service Permissions

Attack flow:

```
Find Service

↓

Check Permissions

↓

Modify Service

↓

Execute as SYSTEM
```

---

## Unquoted Service Paths

Example:

```
C:\Program Files\App\service.exe
```

Windows may search:

```
C:\Program.exe
C:\Program Files\App.exe
```

An attacker can exploit this behavior.

---

## Weak File Permissions

Example:

```
SYSTEM executes file

↓

User can modify file

↓

Replace file

↓

SYSTEM execution
```

---

## Stored Credentials

Search for:

- Configuration files
- Password managers
- Scripts
- Registry credentials

Example:

```cmd
findstr /si password *.txt
```

---

## Token Impersonation

Common privileges:

```
SeImpersonatePrivilege
```

Can be abused with tools such as:

- JuicyPotato
- PrintSpoofer
- RoguePotato

---

## Kernel Exploits

Check:

```
Windows Version

↓

Patch Level

↓

Known Vulnerabilities
```

Search:

```
searchsploit windows kernel
```

- Restricting administrator access
- Securing service permissions
- Using strong passwords
- Monitoring privilege changes
- Auditing ACL permissions
- Limiting user privileges
